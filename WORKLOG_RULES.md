# WorkLog 운영 규칙

이 문서는 Codex에서 업무 기록을 작성할 때 사용하는 공통 규칙이다.  
다른 Codex 스레드에서 업무 기록을 이어갈 때는 이 파일을 먼저 읽고 아래 규칙을 따른다.

## 1. 저장소 정보

- 로컬 저장소: `C:\Users\USER\Desktop\WorkLog`
- 원격 저장소: `https://github.com/Cheol-wook/WorkLog`
- Git 사용자 이름: `Cheol-wook`
- Git 이메일: `loky123@naver.com`
- Git 실행 파일: `C:\Program Files\Git\cmd\git.exe`

## 2. 작업 신호

업무 기록은 사용자가 직접 입력하는 신호를 기준으로 시작하고 종료한다.  
특정 시간에 자동으로 생성하지 않는다.

### 작업 시작

```text
YYYYMMDD 업체명 작업시작
```

예시:

```text
20260707 지비라이트 작업시작
```

### 작업 종료

```text
YYYYMMDD 업체명 작업종료
```

예시:

```text
20260707 지비라이트 작업종료
```

## 3. 업체 폴더 규칙

작업종료 신호를 받으면 저장소 루트에서 업체명과 같은 이름의 폴더를 찾는다.

예시:

```text
C:\Users\USER\Desktop\WorkLog\지비라이트
```

- 업체 폴더가 있으면 해당 폴더를 사용한다.
- 업체 폴더가 없으면 새로 생성한다.
- 새 업체 폴더를 만들 때는 간단한 `README.md`를 함께 생성한다.
- 업체명에 Windows 파일명에서 허용되지 않는 문자가 있으면 임의로 진행하지 말고 사용자에게 안전한 폴더명을 요청한다.

Windows 파일명에서 사용할 수 없는 문자는 다음과 같다.

```text
< > : " / \ | ? *
```

## 4. 생성 파일

작업종료 신호를 받으면 업체 폴더에 아래 두 파일을 생성하거나 갱신한다.

```text
YYYY-MM-DD-internal.txt
YYYY-MM-DD.md
```

예시:

```text
C:\Users\USER\Desktop\WorkLog\지비라이트\2026-07-07-internal.txt
C:\Users\USER\Desktop\WorkLog\지비라이트\2026-07-07.md
```

입력 신호의 `YYYYMMDD` 날짜는 파일명에서 `YYYY-MM-DD` 형식으로 변환한다.

## 5. 내부용 TXT 규칙

`YYYY-MM-DD-internal.txt`는 로컬 전용 내부 기록이다.

포함 가능 내용:

- 테이블명
- DB 구조
- 컬럼명
- 내부 경로
- 구체적인 코드/구현 메모
- 이슈 원인
- 블로커
- 다음 작업
- 실제 작업 중 확인한 기술적 세부사항

이 파일은 GitHub에 올리지 않는다.

## 6. 공개용 MD 규칙

`YYYY-MM-DD.md`는 GitHub에 올리는 공개 이력용 기록이다.

목적:

- 업무 수행 이력 정리
- 이력서/포트폴리오에 활용 가능한 기록
- 공개 가능한 수준의 작업 결과 요약

포함 가능 내용:

- 수행한 작업
- 분석/검토한 범위
- 개선한 점
- 완료 결과
- 다음 작업 방향
- 공개 가능한 기술 키워드

제외해야 할 내용:

- 테이블명
- 내부 DB 구조
- 계정/인증 정보
- 민감한 내부 경로
- 비공개 코드 상세
- 외부 공개가 부적절한 고객사 내부 정보

## 7. GitHub 업로드 규칙

GitHub에는 공개용 MD만 업로드한다.

기본 커밋 대상:

```text
업체명/YYYY-MM-DD.md
```

새 업체 폴더를 만든 경우에는 해당 폴더의 `README.md`도 함께 커밋할 수 있다.

내부용 TXT는 `.gitignore`에 의해 제외한다.

```text
**/*-internal.txt
```

## 8. 커밋 메시지 규칙

업무 기록 커밋 메시지는 아래 형식을 사용한다.

```text
docs: YYYY-MM-DD 업체명 업무 기록
```

예시:

```text
docs: 2026-07-07 지비라이트 업무 기록
```

## 9. 기록 작성 기준

작업종료 신호를 받으면 해당 날짜와 업체명의 가장 최근 작업시작 신호 이후 내용을 기준으로 기록한다.

정리 대상:

- 현재 스레드에서 확인된 작업 내용
- 접근 가능한 관련 Codex 스레드의 작업 내용
- 사용자가 직접 공유한 외부 작업 내용

주의 사항:

- 추측으로 내용을 만들지 않는다.
- 사실로 확인되는 작업, 결정 사항, 완료 결과, 변경/생성 파일, 이슈, 블로커, 다음 할 일만 작성한다.
- 접근 불가능한 외부 채팅방, 메신저, 브라우저 밖 대화는 사용자가 공유한 내용만 반영한다.

## 10. 다른 스레드에서 사용하는 방법

다른 Codex 스레드에서 업무 기록을 진행할 때는 먼저 아래 파일을 읽는다.

```text
C:\Users\USER\Desktop\WorkLog\WORKLOG_RULES.md
```

그 다음 사용자의 작업시작/작업종료 신호에 맞춰 이 문서의 규칙대로 기록을 생성한다.
