# 2026-07-15 개발 기록

## 제조표준서 저장 구조 확장 설계

### 프로젝트

- 제조표준서 관리 기능의 신규 화면 저장 구조 설계 및 기존 흐름 확장

### 수행 작업

- 기존 문서 작성·임시저장·확정·개정 흐름을 기준으로 신규 제조표준서 화면의 데이터 모델을 분석했다.
- 기본정보, 공정 확인, 부모 파트, 상세 구성 데이터를 한 번의 저장 흐름으로 처리하는 구조를 설계했다.
- 화면 데이터와 데이터베이스 입력 형식을 연결하기 위한 테이블형 매개변수 설계를 검토했다.
- 부모 데이터와 상세 데이터가 공통 식별값으로 연결되도록 저장 순서와 중복 검증 기준을 정리했다.
- 고정된 화면 영역은 자동 번호 대신 의미 있는 고정 구분값으로 관리하도록 설계했다.
- 외부 작업 표준을 선택해 화면에 반영하고, 문서 저장 시점에는 해당 내용을 스냅샷으로 보존하는 방식을 정리했다.

### 기술 적용 및 검증

- C# DataTable 기반 데이터 변환
- SQL Server Table-Valued Parameter
- MERGE 기반 임시 저장 Upsert
- 트랜잭션, XACT_ABORT, TRY/CATCH 기반 원자적 저장
- 부모-자식 키 설계 및 중복 데이터 사전 검증
- 문서 유형을 공통 흐름으로 관리하면서 화면별 업무 데이터를 분리하는 방식 검토

### 결과

- 기본정보와 상세 데이터의 저장 순서 및 연결 규칙을 확정했다.
- 화면의 입력값, 선택형 작업 표준, 다중 그리드 데이터를 일관된 저장 모델로 연결할 기반을 마련했다.
- 향후 작업 표준 조회, 화면 바인딩, 확정 및 개정 처리로 확장할 수 있는 구조를 정리했다.
