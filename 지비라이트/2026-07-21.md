## 2026-07-21 업무 기록

### 프로젝트

- 지비라이트 제조표준서 기능 개발

### 수행 작업

- 여러 작업 조건 영역을 하나의 확정 흐름으로 반영하도록 데이터 저장 구조를 확장했다.
- 확정 시 현재 작성본을 이력과 최신 확정본에 일관되게 반영하고, 완료된 임시 데이터를 정리하는 처리 순서를 설계했다.
- 팝업에서 선택한 작업 조건을 화면 데이터 구조에 안정적으로 반영하도록 스키마 병합 방식을 점검했다.
- 확정된 문서를 개정용 작성본으로 복사하는 흐름을 상세 데이터 영역까지 확장했다.
- 개정 팝업과 저장 프로시저의 호출 계약을 점검해 문서번호, 공장 코드, 버전 정보 전달 방식을 정리했다.

### 문제 해결

- 강한 타입 데이터 구조에 조회 결과를 병합할 때 발생한 컬럼 충돌을 분석했다.
- 기존 화면 스키마를 유지하면서 필요한 값만 반영하도록 병합 정책을 조정했다.
- 문서 상태 변경만으로는 문서 이력이 남지 않는 점을 확인하고, 신규 제정 및 개정 확정에 필요한 이력 처리 단계를 도출했다.

### 결과

- 신규 저장, 수정 저장, 상세 조회, 팝업 데이터 반영, 확정 처리의 기본 흐름을 검증했다.
- 개정 시작 시 주요 상세 데이터를 작성용 상태로 전환할 수 있는 기반을 마련했다.
- 문서 이력과 제정 이력 보완 항목을 명확히 정리했다.

### 다음 작업

- 문서 버전 이력과 신규 제정 이력을 확정 처리에 연결한다.
- 개정 후 재확정 시나리오를 통합 테스트한다.
