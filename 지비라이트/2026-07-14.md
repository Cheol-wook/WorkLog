# 2026-07-14 개발 기록

## 제조표준서 개정 흐름 점검

### 작업 목표

- 개정 팝업과 개정 시작 처리 사이의 호출 연결을 검토했다.
- 문서 상태 변경, 개정 메타데이터 등록, 작업 데이터 복사 흐름의 식별값 기준을 점검했다.

### 수행 작업

- 팝업 결과를 기반으로 개정 정보를 전달하는 화면 로직을 검토했다.
- 저장 프로시저의 매개변수 정의와 화면 호출부를 대조해 실행을 막을 수 있는 불일치를 확인했다.
- 개정 시작 시 상태 변경과 개정 이력 정보가 동일한 문서를 기준으로 연결되어야 하는 점을 정리했다.
- 입력값의 형식과 특수문자 처리 방식을 점검했다.

### 결과

- 개정 시작 호출부의 수정 대상과 검증 순서를 명확히 했다.
- 문서 식별값을 일관되게 관리해야 이후 이력 조회와 확정 처리가 정확해진다는 연결 기준을 확인했다.

### 다음 작업

- 개정 시작 기능을 실제 데이터로 검증한다.
- 개정 요청과 확정 시점의 연결 정보를 보완한다.
- 화면 데이터 변경 상태 관리와 확정 처리 오류를 점검한다.
