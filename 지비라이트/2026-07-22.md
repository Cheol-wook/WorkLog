### 프로젝트

- 지비라이트 제조표준서 문서 생명주기 및 화면 동작 개선

### 수행 작업

- 신규 제정과 개정 확정 시 문서 상태, 버전, 이력을 일관되게 관리하는 처리 흐름을 설계했다.
- 확정 시 최신 문서 정보와 버전별 이력을 함께 남기고, 최초 제정과 개정의 요청 이력 생성 시점을 구분했다.
- 개정 시작 시 최신 확정본을 작성용 데이터로 전환하는 범위를 전체 상세 영역까지 점검했다.
- 화면 선택값 변경에 따른 목록 갱신 이벤트와 데이터 병합 방식을 검토했다.

### 문제 해결

- 업무 데이터 이력만 생성되고 문서 이력이 누락되는 구조를 분석했다.
- 최초 확정과 개정 확정에서 서로 다른 버전 및 요청 이력 규칙이 필요함을 확인했다.
- 화면 데이터 필터링 과정에서 발생할 수 있는 스키마 및 상태 관리 문제를 예방하도록 처리 방식을 정리했다.

### 결과

- 문서 상태, 버전, 요청 이력, 업무 이력을 하나의 확정 흐름으로 연결할 수 있는 구현 방향을 마련했다.
- 개정 시작 시 주요 상세 데이터를 유지하는 구조를 확인했다.
- 콤보박스 변경에 반응하는 화면 데이터 갱신 기준을 정리했다.

### 다음 작업
