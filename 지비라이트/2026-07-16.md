# 2026-07-16 개발 기록

## 제조 기준 데이터 저장 흐름 확장

### 프로젝트

- 지비라이트 제조표준서 관리 기능

### 작업 목표

- 외부 기준 데이터를 선택해 제조 기준서의 여러 상세영역으로 일관되게 저장하는 흐름을 설계했다.

### 수행 작업

- 기본정보, 부모 항목, 상세 그리드 데이터를 하나의 저장 요청으로 전달하는 구조를 점검했다.
- 설비별 기준정보, 설정값, 구간별 속도, 조건별 롤 설정을 계층형 데이터 모델로 분리했다.
- 화면의 세 그리드 데이터를 각각의 저장 형식으로 변환하는 DataTable 기반 매핑을 구성했다.
- 상단과 하단 설정값을 별도 데이터로 저장하도록 식별값과 실제 설정값의 역할을 분리했다.
- 복수 데이터 묶음을 Table-Valued Parameter로 전달하고, 단일 저장 프로시저에서 원자적으로 처리하는 방식을 정리했다.

### 기술 적용 및 검토

- C# DataTable 기반 데이터 변환
- SQL Server Table-Valued Parameter
- MERGE 기반 임시 저장 Upsert
- 부모-자식 키 설계 및 저장 순서 관리
- 트랜잭션 기반 다중 데이터 저장

### 결과

- 설비 기준정보와 세 개의 상세 그리드를 공통 식별값으로 연결하는 저장 구조를 마련했다.
- 기준 데이터 선택 결과에서 표시값과 내부 식별값을 구분해 저장할 수 있도록 설계를 보완했다.
- 신규 작성과 작성 중 수정에서 같은 저장 경로를 사용할 수 있는 기반을 확장했다.

### 다음 작업

- 선택 팝업의 내부 식별값 전달을 보완한다.
- 빈 상세영역을 포함한 저장 시나리오를 점검한다.
- 실제 임시 저장과 이후 확정 처리 흐름을 통합 검증한다.
