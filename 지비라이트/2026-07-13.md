# 2026-07-13 개발 기록

## 제조표준서 조회 및 개정 이력 설계

### 작업 목표

- 제조표준서 조회 화면의 표시 정보를 보완하고, 문서 이력 조회 구조를 설계했다.
- 확정본과 현재 수정 중인 문서를 한 화면에서 구분해 다룰 수 있는 기준을 정리했다.

### 수행 작업

- 코드값으로 저장된 생산 단계와 품목 정보를 화면용 명칭으로 변환하는 조회 구조를 검토했다.
- 기본정보 조회에서 관련 품목명과 분류 정보를 함께 가져오는 조인 방식을 정리했다.
- 여러 화면에서 재사용할 수 있도록 문서 유형, 문서 번호, 참조값을 조건으로 받는 공용 개정 이력 조회 구조를 설계했다.
- 확정 이력과 진행 중인 개정 초안을 하나의 목록에 표시하되, 상태 구분값으로 서로 다른 상세 조회 경로를 선택하도록 구성했다.
- 상태 코드를 공통 코드 테이블의 표시명으로 변환하는 방식을 적용했다.

### 설계 결정

- 확정 이력은 변경 시점의 스냅샷 데이터를 기준으로 조회한다.
- 진행 중인 개정은 확정 이력과 구분된 현재 작업 행으로 표시한다.
- 이력 행 선택 시에는 과거 확정본을 읽기 전용으로 열고, 진행 중 행 선택 시에는 현재 작성 데이터를 연다.
- 이력 목록에는 화면 표시용 정보와 함께 상세 조회에 필요한 식별값을 숨김 컬럼으로 유지한다.

### 검토 사항

- 문서 이력, 개정 요청 정보, 실제 업무 데이터 스냅샷의 역할을 분리해 데이터 흐름을 재정리했다.
- 문서 식별값과 개정 시점이 서로 다른 테이블에서 일관되게 연결되어야 과거 상세 조회가 정확하게 동작한다는 점을 확인했다.
- 확정 처리 중 변경사항 감지 오류 가능성을 확인했으며, 다음 단계에서 화면 데이터 상태 관리 흐름을 점검하기로 했다.

### 다음 작업

- 공용 개정 이력 조회를 화면에 연결하고 이력 행 선택 기능을 구현한다.
- 과거 확정본과 현재 수정본의 상세 조회를 각각 연결한다.
- 확정 과정의 변경사항 감지와 데이터 상태 초기화 로직을 점검한다.
