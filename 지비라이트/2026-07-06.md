# 2026-07-06 개발 기록

## 제조표준서 등록 기능 설계 및 구현

제조표준서 작성 화면에서 여러 영역의 데이터를 한 번에 저장할 수 있도록 C#과 SQL Server 저장 구조를 설계하고 구현했다.

### 수행 내용

- 신규 작성, 작성 중 수정, 확정, 개정으로 이어지는 문서 생명주기 정리
- 임시 데이터, 현재 데이터, 개정 이력을 분리하는 저장 구조 검토
- 여러 그리드의 데이터를 공통 DataTable로 통합하는 C# 변환 로직 작성
- 부모 파트와 상세 데이터 사이의 키 매핑 및 저장 순서 설계
- 고정 파트와 사용자 선택형 파트를 동일한 공통 함수로 처리하도록 구조화
- Table-Valued Parameter를 사용하여 여러 데이터 묶음을 한 번의 프로시저 호출로 전달
- `MERGE`를 활용해 최초 등록과 작성 중 수정을 하나의 저장 흐름으로 통합
- 트랜잭션과 TRY/CATCH를 적용해 일부 데이터만 저장되는 문제 방지
- 중복 키, 부모 데이터 존재 여부, NULL 및 숫자 타입 변환 가능성 검토
- SQL 편집기의 `THROW` 구문 파싱 문제를 분석하고 문장 종료 방식으로 해결

### 주요 기술

- C# / DataTable
- SQL Server / T-SQL
- Table-Valued Parameter
- MERGE 기반 Upsert
- Transaction / TRY-CATCH / XACT_ABORT
- 부모-자식 데이터 모델링

### 결과

- 기본정보와 여러 상세영역을 일관된 문서번호로 저장하는 기반 완성
- 여러 화면 그리드를 부모 파트와 정확히 연결하는 공통 패턴 확립
- 신규 등록과 임시 수정 과정을 하나의 원자적 저장 작업으로 통합
- 이후 추가 상세영역과 확정·개정 기능을 같은 패턴으로 확장할 수 있는 구조 마련
