## 2026-07-23 업무 기록

### 프로젝트

- 지비라이트 제조표준서 및 배합표의 데이터 일관성과 개정 이력 조회 기능 개선

### 수행 작업

- 상세 그리드 저장 과정에서 식별값과 표시값이 혼용되지 않도록 데이터 매핑 흐름을 점검했다.
- 참조 무결성 오류를 분석해 화면 데이터, 조회 결과, 저장 데이터 간의 식별값 전달 기준을 정리했다.
- 조회 시에는 사람이 읽을 수 있는 이름을 표시하고, 저장 시에는 안정적인 식별코드를 유지하는 방식을 설계했다.
- 일반 제조표준서의 개정 이력 선택 시 기본정보와 여러 상세 영역을 해당 버전 기준으로 복원하는 조회 구조를 구성했다.
- 공통 개정 이력 목록의 상태 구분값과 기준일자를 활용해 확정 이력과 현재 작성 데이터를 분기하는 화면 흐름을 정리했다.
- 배합표에도 확정 이력과 현재 개정중 데이터를 구분해 불러오는 구조를 설계했다.

### 문제 해결

- 화면 표시용 명칭이 저장 식별값으로 전달되거나 빈 값으로 전달될 수 있는 경로를 발견했다.
- 참조 제약 오류의 원인을 데이터베이스 저장 로직이 아닌 화면 데이터 매핑 단계로 좁혔다.
- 기존 이력 조회에서 선택 행의 기준값과 조회 조건이 일치하지 않아 결과가 반환되지 않는 문제를 분석했다.
- 개정 이력 클릭 시 다른 화면 목록의 행 번호를 잘못 사용하는 가능성을 찾아, 선택된 이력과 현재 문서의 역할을 분리했다.

### 결과

- 저장용 식별코드와 조회용 표시명 분리 원칙을 일반 및 산포 제조표준서에 적용할 기준을 마련했다.
- 확정본과 현재 개정본을 각각의 데이터 상태에 맞게 조회하는 공통 패턴을 정리했다.
- 제조표준서와 배합표의 개정 이력 조회 기능을 동일한 문서 생명주기 기준으로 확장할 수 있게 됐다.

### 다음 작업

- 식별코드 매핑 수정 후 저장 및 재조회 시나리오를 검증한다.
- 개정 이력 그리드 선택 시 확정본과 현재 개정본이 올바르게 복원되는지 화면 테스트를 진행한다.
- 입력값 검증을 추가해 필수 데이터 누락을 저장 전에 차단한다.
